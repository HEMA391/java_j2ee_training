    • Working of a Router Component

  • Routes is NOTHING but HYPERLINKS linked to Angular COMPONENT
  • Its NOT JUST hyperlink, a component also get INITIALISED to handle the URL
  • app.component.html is the 1st page to load
    In app.component.html, note the usage of router-outlet tag
  • router-outlet then loads the app.routes.ts
  • The routing paths are loaded using app.routes.ts
    viz, see the keyword redirectTo in the above file
    If there is no PATH specified, the default path taken is products

    Simple FLOW

  • app.component.html--calls-->router-outlet---calls-->app.route.ts--loads-->the component---renders--->html

    Component and Routing for adding new Product

  • Create a new path in app.routes.ts, viz, new_product
  • Create a new component in command prompt: ng g component new_product
  • In app.routes.ts, Import and Point the component as->component: NewProductComponent

    Testing the new routing path

  • Simply change the URL in the browser to http://localhost:4200/new_product
  • You should be able to see the html content from new-product/

    • Product Component [product.ts]

  • Represents a Product

    ProductService [vendor.service.ts]

  • This creates an array of objects using Observable of RxJs
  • Is Injected

    ProductListComponent [product-list\product-list.component.ts]

  • Calls the ProductService and initialises product as array

    ProductDetailsComponent [product-details\product-details.component.ts]

  • Represent a specific product

    • Vendor Component [vendor.ts]

  • Represents a Vendor

    VendorService [vendor.service.ts]

  • This creates an array of objects using Observable of RxJs
  • Is Injected

    VendorListComponent [vendor-list\vendor-list.component.ts]

  • Calls the VendorService and initialises vendor as array

    VendorDetailsComponent [vendor-details\vendor-details.component.ts]

  • Represent a specific Vendor

    • Creating the PRODUCT Component

  • Navigate to the directory D:\ws\angular2
  • Create a new project and its skeleton

    • ng new vendorportal
  • Download all files from the Git
    Use ONLY for reference
  • Launch the app

    • cd vendorportal
    ng serve –open
  • If you get error like Cannot find module '@angular-devkit/core', navigate to D:\ws\angular\observable
    Update npm: npm update -g @angular/cli
    npm i -D @angular-devkit/core
  • Create Service for Product
    In vendorportal folder, ng g service product
    Do NOT give service name as product.service because the command itself create service name as product.service
  • Create Class to represent Vendor
    In vendorportal folder, ng g class product
    add attributes like id, name etc
    Update code as per the tutorial
  • Create Component to list Product
    In vendorportal folder, ng g component product-list
    Update code as per the tutorial, if you could not do yourself
  • Update app.module.ts to include ProductService and ProductListComponent
  • Update app.routes.ts to have /products as default route
  • Update title in app.component.ts
