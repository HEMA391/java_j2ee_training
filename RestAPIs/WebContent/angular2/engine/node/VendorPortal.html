Vendor Management Sample with Observable, Router and Service Sample

Pre-requisite:
  • VendorController in SpringBootMaven project should be running
  • Observable
  • Routers
  • Service
  • Injectable

    • Person Component [person.ts]

  • Person is a basic object representing a person

    PeopleService [people.service.ts]

  • This creates an array of objects using Observable of RxJs
  • Is Injected

    PeopleListComponent [people-list\people-list.component.ts]

  • Calls the PeopleService and initialises people as array

    PersonDetailsComponent [people-details\people-details.component.ts]

  • Represent a specific Person

  • Step by Step Guide: barbarianmeetscoding.com
  • Navigate to the directory D:\ws\angular2
  • Create a new project and its skeleton

    • ng new vendorportal
  • Download all files from the Git
  • Navigate to D:\ws\angular2\vendorportal\src\app directory

    • Replace the contents of this directory with the files downloaded
  • Launch the app

    • cd vendorportal
    ng serve –open
  • If you get error like Cannot find module '@angular-devkit/core', navigate to D:\ws\angular\observable
    Update npm: npm update -g @angular/cli
    npm i -D @angular-devkit/core
  • Create Service for Vendor
    In vendorportal folder, ng g service vendor
    Do NOT give service name as vendor.service because the command itself create service name as vendor.service
  • Create Class to represent Vendor
    In vendorportal folder, ng g class vendor
    add attributes like id, name etc
    Update code as per the tutorial
  • Create Component to list Vendor
    In vendorportal folder, ng g component vendor-list
    Update code as per the tutorial
  • Update app.module.ts to import the VendorList Component
    Remove the PeopleListComponent and PeopleService
  • Update app.routes.ts to import and declare the VendorList Component
    Remove the PeopleListComponent and PeopleService if PRESENT
